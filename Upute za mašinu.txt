Namespace

U Dashboardu gore lijevo imaš namespace dropdown.

Deployment

U Dashboardu klikneš gore desno plavi +.

Zalijepiš YAML:

apiVersion: apps/v1
kind: Deployment
metadata:
  name: hello-kube
  namespace: do188-chapter8
spec:
  replicas: 1
  selector:
    matchLabels:
      app: hello-kube
  template:
    metadata:
      labels:
        app: hello-kube
    spec:
      containers:
        - name: nginx
          image: docker.io/library/nginx:latest
          ports:
            - containerPort: 80

Nakon toga provjeriš:

Workloads → Deployments

Trebaš vidjeti:

hello-kube
Pods 1/1

Zatim:

Workloads → Pods

Pod mora biti:

Running
Service za Deployment

Opet plavi +.

Zalijepiš:

apiVersion: v1
kind: Service
metadata:
  name: hello-kube
  namespace: do188-chapter8
spec:
  type: NodePort
  selector:
    app: hello-kube
  ports:
    - protocol: TCP
      port: 80
      targetPort: 80

Bitno: kod ports: mora biti crtica:

ports:

ports:

6. Što moraš znati objasniti

Ovo je najvažnije. Naredbe su manje bitne ako ne znaš objasniti logiku.

Podman
Podman lokalno pokreće containere na jednoj mašini.

Primjeri:

podman run
podman ps
podman logs
podman exec
podman build
podman compose up -d
podman compose down
Kubernetes
Kubernetes ne pokrećeš ručno container po container.
Kubernetesu opišeš željeno stanje.

Lanac:

Deployment → ReplicaSet → Pod → Container

Service lanac:

Service → selector → Endpoint → Pod

Primjer:

Service selector app=do180-nginx
traži Pod koji ima label app=do180-nginx
i šalje promet na njega.
7. Najčešće greške koje si imao / koje trebaš izbjeći
Greška 1 — miješanje Podmana i Kubernetes Dashboarda

Podman nije u Dashboardu.

Podman = lokalni CLI alat
Kubernetes Dashboard = UI za Kubernetes objekte
Greška 2 — krivi YAML za Service ports

Krivo:

ports:
  protocol: TCP
  port: 80
  targetPort: 80

Ispravno:

ports:
  - protocol: TCP
    port: 80
    targetPort: 80
Greška 3 — rad u krivom namespaceu

Uvijek prvo provjeri gore u Dashboardu namespace:

do188-chapter8

ili:

do180-chapter1

Nemoj bezveze raditi u:

default
kube-system
kubernetes-dashboard
Greška 4 — očekivati da je Pod direktno dostupan izvana

Pod IP je interni.

Zato trebaš Service:

Pod IP = interno
Service = stabilni pristup
NodePort = pristup izvana preko node IP-a i porta

Kod tebe:

Minikube IP: 192.168.49.2
hello-kube NodePort: 31118
do180-nginx NodePort: 32451

Test:

curl http://192.168.49.2:31118
curl http://192.168.49.2:32451
